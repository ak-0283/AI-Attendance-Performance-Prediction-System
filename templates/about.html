About The AI Attendance System

Data-driven insights to identify student performance risk early and support timely intervention.

Try Prediction View Results

Mission

Our goal is to translate attendance and academic signals into clear, actionable risk categories — Safe, At Risk, and Critical — so educators can intervene before outcomes decline.

System Architecture

  • Frontend: Flask templates in templates/ with clean, responsive CSS under static/style.css.
  • Backend: Python Flask app (`app.py`) serving routes for home, dataset, model, results, predict, and contact.
  • Model: Trained classifier saved in model/, loaded for real-time predictions on the Predict page.
  • Artifacts: Training metrics and report saved in static/model_report.txt for transparency.

Model Details

The project currently uses a robust tree-based ensemble (e.g., Random Forest) to classify risk. Key signals include attendance consistency, study time, past failures, parental support, and relevant demographic/academic features.

  • Target: Performance risk band prediction.
  • Inputs: Attendance frequency, study habits, academic history, support variables.
  • Training Script: train_model.py — reproducible, easy to re-run.
  • Evaluation: Precision, recall, F1, confusion matrix (see Results).

Dataset

The system uses curated student datasets including data/student-mat.csv and supplemental records in data/student_data.csv.

  • Cleaning: Handle missing values, normalize categorical fields, and align schemas.
  • Feature Engineering: Derive attendance rates and composite study indicators.
  • Reproducibility: Retrain anytime with python train_model.py.
