Dataset

Overview

This app uses curated student datasets to understand how attendance and related academic factors correlate with performance risk. The primary source is the well-known student performance dataset.

Files in Use

Core Dataset

data/student-mat.csv

Base academic records used for training and evaluation.

Supplementary Records

data/student_data.csv

Additional fields aligned to the core dataset for experiments.

Feature Highlights

  • Attendance consistency and frequency indicators.
  • Study time and past academic performance signals.
  • Support variables (e.g., family/guardian, school engagement).
  • Derived features engineered during preprocessing.

Preprocessing Checklist

  • Handle missing values and normalize categorical encodings.
  • Align schema across files and remove duplicate entries.
  • Create attendance rate features and composite indices.
  • Split train/test with consistent random seed for reproducibility.

Sample Columns

Column Description Type
attendance_rate Derived ratio indicating attendance consistency. Numeric
study_time Weekly hours spent studying. Numeric
past_failures Count of prior course failures. Numeric
support_flags Indicators for academic support (school/family). Categorical

Privacy & Responsible Use

  • Store datasets securely and use anonymized identifiers where possible.
  • Predictions assist decisions; they do not replace educator judgment.
  • Retrain periodically to reflect new cohorts and data distributions.

Quick Steps

  1. Review data files listed above.
  2. Retrain the model with python train_model.py.
  3. Inspect evaluation on the Results page.
  4. Use Predict to score new profiles.
