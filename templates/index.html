AI Attendance & Performance Prediction System

An end-to-end web app that analyzes student data and attendance trends to predict academic risk levels — Safe, At Risk, or Critical — so you can take timely, targeted action.

What This System Does

  • Transforms raw student records into meaningful predictors of performance.
  • Classifies each student into risk bands: Safe, At Risk, Critical.
  • Highlights attendance patterns correlated with performance outcomes.
  • Supports early intervention with data-backed insights.

How It Works

  1. Data: Uses the curated dataset in data/student-mat.csv along with additional student records like data/student_data.csv.
  2. Features: Attendance frequency, study time, past failures, family support, and other academic/context variables.
  3. Model Training: Run train_model.py to train and evaluate the classifier; reports are saved to static/model_report.txt.
  4. Prediction: The trained model (stored under model/) powers the Predict page for on-demand scoring.

Key Features

  • Attendance-aware insights: Quantifies how attendance impacts predicted performance risk.
  • Simple web UI: Navigate via the top menu; no complex setup needed.
  • Reproducible training: One command to re-train and refresh results.
  • Transparent metrics: See precision/recall/F1 and confusion matrix on the Results page.
  • Actionable categories: Risk bands mapped to suggested interventions.

Quick Start

  1. Open the Dataset page to review inputs.
  2. Train or refresh the model by running python train_model.py in your workspace.
  3. Check evaluation details on Results.
  4. Use Predict to score a student profile.

Risk Bands & Suggested Actions

  • Safe: Maintain current study plan; periodic check-ins.
  • At Risk: Increase attendance consistency; add weekly tutoring or mentoring.
  • Critical: Immediate academic support plan; attendance contract; parental engagement.

Model & Data Transparency

Training artifacts and performance reports are kept within the project for auditability:

  • model/: Serialized trained model and related files.
  • static/model_report.txt: Latest metrics snapshot after training.
  • data/student-mat.csv: Base dataset used for experiments.

Privacy & Responsible Use

  • Use anonymized identifiers when possible and store data securely.
  • Predictions assist decisions; they do not replace educator judgment.
  • Retrain periodically to keep the model aligned with new cohorts.

FAQ

Does this require internet? No, all processing runs locally in this app.

Can I add new features? Yes — extend feature engineering in train_model.py and retrain.

Where do I see accuracy? Open the Results page or check static/model_report.txt.
